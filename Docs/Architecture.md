
# Implementation of a 16-Stage Pipelined DES Hardware Accelerator with First-Order Boolean Masking for SCA Resilience

This diagram illustrates the hardware data path and key schedule as implemented in Verilog. 
It uses standard hardware block representations.

```text
========================================================================================================
                          TOP LEVEL I/O & PRE-PROCESSING (des_top.sv)
========================================================================================================
 
  Plaintext (64)          Mask In (64)                           Key (64)
       │                       │                                     │
       │     ┌─────────┐       ├─────────────────────────┐           │
       └────►│   XOR   │◄──────┘                         │           ▼
             └────┬────┘                                 │        [ PC-1 ] (Parity drop)
                  │ P_masked = P ^ Mask_In               │           │
                  ▼                                      ▼           │ 56-bit Key
            [ IP (Data) ]                          [ IP (Mask) ]     │
                  │                                      │           │
           ┌──────┴──────┐                        ┌──────┴──────┐    ▼ 
           │             │                        │             │  [KEY SCHEDULE]
    L0_masked (32)   R0_masked (32)            ML0 (32)      MR0 (32)  (Parallel)


========================================================================================================
                             PIPELINE STAGE 1 (des_round.sv)
========================================================================================================
           │             │                        │             │           │
           │             └───────────────────┐    │             └─────┐     │
           │                                 │    │                   │     │ K1 (48-bit subkey)
           │     ┌───────────────────────────┼────┼───────────────────┼─────┼───[ Shift & PC-2 ]
           │     │   ┌───────────────────────┼────┼───────────────────┼─────┘
           │     │   │                       │    │                   │
           │     │   │   +-------------------│----│-------------------+  
           │     │   │   |                   ▼    ▼                   |  
           │     │   │   |  [ MASKED FEISTEL FUNCTION (feistel.sv) ]  |  
           │     │   │   |                                            |
           │     │   │   |   R_masked        MR_in         K1         |
           │     │   │   |      │              │            │         |
           │     │   │   |      ▼              ▼            │         |
           │     │   │   |  [ E-Box ]      [ E-Box ]        │         |
           │     │   │   |      │              │            │         |
           │     │   │   |      └─────►(XOR)◄───────────────┘         |
           │     │   │   |               │     │                      |
           │     │   │   |               ▼     ▼                      |
           │     │   │   |      [ 8 x Boolean Masked S-Box ]◄─(Re-Mask Value generated from MR_in)
           │     │   │   |               │     │                      |
           │     │   │   |               ▼     ▼                      |
           │     │   │   |            [ P-Box ]                       |
           │     │   │   |               │     │                      |
           │     │   │   +---------------│-----│----------------------+  
           │     │   │                   │     │  
           │     │   └────────►(XOR)◄────┘     │  
           │     │               │             │  
           │     │               │             └──────►(XOR)◄────┐
           │     │               │                       │       │
           ▼     ▼               ▼                       ▼       ▼
       [ REG ] [ REG ]       [ REG ]                 [ REG ]   [ REG ]  <-- Pipeline Registers (clk)
          │       │             │                       │         │
      L1_mask  ML1_mask      R1_mask                 MR1_mask  (to next subkey...)
          │       │             │                       │
          │       └─────────────┼───────────────────────┤ (Passed to Stage 2's L_in and ML_in)
          └─────────────────────┘                       │


========================================================================================================
                             PIPELINE STAGES 2 to 15 ...
========================================================================================================
                        (Identical architecture to Stage 1)


========================================================================================================
                             PIPELINE STAGE 16 (des_round.sv)
========================================================================================================
           │       │             │                       │
         [ REG ] [ REG ]       [ REG ]                 [ REG ]
          │       │             │                       │
      L16_mask  ML16_mask    R16_mask                MR16_mask


========================================================================================================
                          TOP LEVEL POST-PROCESSING (des_top.sv)
========================================================================================================
          │       │             │                       │
          │       │             │                       │
          │       └─────────────┼───────────────────────┼──────┐
          └─────────────────────┼─────────┐             │      │
                                │         │             │      │
                         (32)   ▼  (32)   ▼      (32)   ▼  (32)▼
                        [ R16_mask, L16_mask ]   [ MR16_mask, ML16_mask ]  <-- 32-bit Swap
                                  │                       │
                                  ▼                       ▼
                           [ IP_INV (Data) ]       [ IP_INV (Mask) ]
                                  │                       │
                                  └───────►(XOR)◄─────────┘  <-- Final Unmasking
                                             │
                                             ▼
                                     Ciphertext (64-bit)
```
